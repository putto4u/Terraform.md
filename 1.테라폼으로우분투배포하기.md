내 PC에 VirtualBox(버추얼박스)가 설치되어 있으면, Terraform을 이용해서 Ubuntu VM을 자동으로 배포할 수 있습니다. 아래 단계별로 과정을 안내합니다.[2]

### 1. 필요한 프로그램 설치
- VirtualBox: 이미 설치되어 있음
- Terraform: [Terraform 공식 홈페이지](https://developer.hashicorp.com/terraform/install?product_intent=terraform)에서 운영체제에 맞는 바이너리 설치.[2]

### 2. VirtualBox Provider 설치 및 설정
Terraform이 VirtualBox를 관리하려면 community plugin(vbox provider)이 필요합니다. 공식 플러그인은 아직 없으며, community github 저장소에서 `.zip` 등으로 직접 다운받아 plugins 폴더에 넣으면 됩니다.

### 3. Terraform 프로젝트 예시 파일 구조
- `main.tf`: VM 정의
- `variables.tf`: 변수 정의(선택)
- `outputs.tf`: 결과 출력(선택)
- `terraform.tfvars`: 변수 입력(선택)

#### main.tf 예시 (커뮤니티 vbox provider 기준)
```hcl
terraform {
  required_providers {
    virtualbox = {
      source = "terra-farm/virtualbox"
      version = "0.2.1"
    }
  }
}

provider "virtualbox" {}

resource "virtualbox_vm" "ubuntu" {
  name   = "ubuntu-vm"
  os_type = "Ubuntu_64"
  cpu    = 2
  memory = 2048
  disk {
    size = 20000
  }
  iso = "<우분투-ISO-파일-경로>"   # 예시: "/home/user/Downloads/ubuntu-22.04.iso"
  # ssh provisioning 등 추가 가능
}
```

### 4. VM에 한국어/아시아/서울 타임존 설정 옵션
- Ubuntu 설치 후 `locale`과 `timezone`을 변경하는 프로비저닝 스크립트(SSH 등의 자동화 스크립트)를 Terraform 리소스에 붙일 수 있음.
- 예시 스크립트:
```sh
sudo timedatectl set-timezone Asia/Seoul
sudo apt-get install -y language-pack-ko
sudo update-locale LANG=ko_KR.UTF-8
```

### 5. 적용 방법
1. 터미널에서 Terraform 프로젝트 디렉터리 진입
2. `terraform init` 명령 실행
3. `terraform apply` 실행 —> VM 자동 생성

***

#### 참고
- Windows, Linux, Mac 별 플러그인 설치 위치와 terraform 플러그인 폴더 위치는 운영체제마다 다를 수 있음.
- 커뮤니티 VirtualBox provider는 공식 문서와 github(예: https://github.com/terra-farm/terraform-provider-virtualbox) 참고

### 추가 팁
- 완전 자동화를 원하면 Ubuntu ISO가 `unattended` 프리셋(자동응답설정) 파일까지 지원해야 vm 사용자 설정과 ssh 활성화, 언어 타임존까지 한 번에 배포할 수 있음.

***
